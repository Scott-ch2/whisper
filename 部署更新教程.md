# Whisper 语音翻译系统 — 新电脑部署 & 更新教程

## 一、首次部署（新电脑从头搭建）

### 第 1 步：安装 Docker Desktop

| 系统 | 下载地址 |
|------|---------|
| Windows | https://www.docker.com/products/docker-desktop/ |
| macOS | https://www.docker.com/products/docker-desktop/ |

安装后打开 Docker Desktop，确认右下角/顶部状态栏 Docker 图标亮起。

### 第 2 步：打开终端

- **Windows**：按 `Win + X` → 选择 **Windows PowerShell (管理员)**
- **Mac**：打开 **终端.app**

### 第 3 步：拉取项目代码（只要这一条命令）

```powershell
git clone https://github.com/Scott-ch2/-.git whisper
```

解释：`-.git` 是实际的仓库地址，`whisper` 是保存到本地的文件夹名。

### 第 4 步：进入目录并启动

```powershell
cd whisper
docker compose up -d --build
```

然后等 8~20 分钟，完成时会显示三个容器都变成 `Up` 状态。

### 第 5 步：访问系统

打开浏览器 → `http://localhost`

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | admin | admin123 |
| 普通用户 | test | 123456 |

## 二、日常更新（代码有修改后）

如果你在旧电脑上改了代码并推送到了 GitHub，新电脑上只需：

```powershell
cd C:\Users\Lenovo\Desktop\whisper

# 拉取最新代码
git pull

# 重新构建并启动（Docker 会自动缓存不变的部分，只需 1~3 分钟）
docker compose up -d --build
```

## 三、常用命令速查

| 用途 | 命令 |
|------|------|
| 启动全部服务 | `docker compose up -d` |
| 查看运行状态 | `docker compose ps` |
| 查看日志 | `docker compose logs -f` |
| 只看后端日志 | `docker compose logs -f backend` |
| 重启服务 | `docker compose restart` |
| 停止服务 | `docker compose down` |
| 停止并删数据 | `docker compose down -v` |

## 四、常见问题

**Q：提示 "git 不是内部或外部命令"**
→ 去 https://git-scm.com/downloads 下载安装 Git，装完重新打开终端。

**Q：提示 "端口被占用"**
→ 编辑 `docker-compose.yml`，把 `"80:80"` 改成 `"8080:80"`，然后访问 `http://localhost:8080`

**Q：docker compose up 后一直卡住**
→ 网络问题，按 `Ctrl+C` 取消，重新执行一次即可。
